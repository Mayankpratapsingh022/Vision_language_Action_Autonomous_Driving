# Urban VLA Expert v1

Urban VLA Expert v1 is a simulator dataset for language-conditioned urban driving. Each frame pairs a 256 x 256 front-camera image with ego state, a natural-language instruction, and continuous driving controls.

This is a small research dataset, not evidence that a policy is ready for a real vehicle. The expert is a deterministic simulator controller, and the language prompts are curated paraphrases rather than speech collected from drivers.

## What is included

| Group | Episodes | Frames | Hours | Use |
| --- | ---: | ---: | ---: | --- |
| Train | 756 | 224,133 | 6.23 | Behavior cloning |
| Validation | 162 | 47,614 | 1.32 | Model selection |
| Test | 162 | 47,933 | 1.33 | Final open-loop evaluation |
| Failure analysis | 90 | 2,757 | 0.08 | Safety analysis only |
| **Total** | **1,170** | **322,437** | **8.96** | |

The 1,080 expert episodes contain 900 nominal drives and 180 controlled recoveries. Recovery episodes begin from conditions such as a lateral offset, heading error, overspeed, or a stalled start, then use the expert to return to a valid trajectory. The 90 failure episodes are deliberately unsafe cases. They are kept under `raw/failures/` and must not be mixed into the imitation-learning target data.

The nine driving intents are:

- turn left at an intersection
- continue straight through an intersection
- turn right at an intersection
- obey traffic lights
- stop for pedestrians
- pass a slow vehicle
- yield to a cut-in vehicle
- follow a curved road
- take a detour around a blocked lane

There are no roundabout scenarios in this version.

## Files

```text
urban-vla-expert-v1/
|-- config.yaml
|-- schema.json
|-- manifests/
|   |-- nominal.jsonl
|   |-- recovery.jsonl
|   `-- failures.jsonl
|-- raw/
|   |-- accepted/<episode-id>/
|   |   |-- episode.json
|   |   |-- telemetry.jsonl
|   |   `-- front.mp4
|   `-- failures/<episode-id>/
|       |-- episode.json
|       |-- telemetry.jsonl
|       `-- front.mp4
`-- reports/validation-summary.json
```

`episode.json` stores the instruction, task ID, split, random seeds, weather, traffic density, expert profile, and outcome. `telemetry.jsonl` has one row per video frame. The MP4 and telemetry both run at 10 Hz.

## Observation and action

The state vector is:

```text
[speed_mps, steering, previous_throttle, previous_brake]
```

The action vector is:

```text
[throttle, brake, steering]
```

Throttle and brake are in `[0, 1]`. Steering is in `[-1, 1]`. The telemetry also records route progress, route error, heading error, collision state, and whether the expert is recovering.

## Download

```bash
hf download Mayank022/urban-vla-expert-v1 \
  --repo-type dataset \
  --local-dir datasets/urban-vla-expert-v1
```

Python code can fetch the same snapshot:

```python
from huggingface_hub import snapshot_download

dataset_root = snapshot_download(
    repo_id="Mayank022/urban-vla-expert-v1",
    repo_type="dataset",
)
print(dataset_root)
```

Use the manifest `split` field instead of making a fresh random split. The supplied split keeps world seeds and instruction paraphrases from leaking across train, validation, and test.

## Reading an episode

```python
import json
from pathlib import Path

root = Path(dataset_root)
episode_dir = next((root / "raw" / "accepted").iterdir())

metadata = json.loads((episode_dir / "episode.json").read_text())
telemetry = [
    json.loads(line)
    for line in (episode_dir / "telemetry.jsonl").read_text().splitlines()
    if line
]

print(metadata["episode"]["instruction"])
print(telemetry[0]["observation"]["state"])
print(telemetry[0]["action"])
```

Decode `front.mp4` with PyAV, OpenCV, or FFmpeg. The current VLA workflow converts the separate human left-turn recordings into LeRobot v3 before fine-tuning.

## Collection and checks

The episodes were generated by the `vla-urban-autonomy-sim` automated expert. Weather, traffic density, route seeds, traffic seeds, driving profile, instruction wording, and recovery perturbations vary across the collection.

The final validation checked manifest counts, episode completeness, metadata and telemetry shape, video readability, frame counts, action bounds, split leakage, and blank frames. The published report has `valid: true` with no errors or warnings.

## Limits

- Everything is synthetic. Camera appearance and vehicle dynamics do not match a particular real car.
- The policy targets come from a rules-based expert, so the dataset carries that controller's habits and blind spots.
- Language coverage is narrow and written in English.
- Open-loop action error is useful for debugging but does not replace closed-loop simulator evaluation.
- Failure episodes show unsafe behavior. They are analysis data, not negative targets for standard behavior cloning.

Do not use a model trained on this dataset to control a real vehicle.
